Windows 自动备份 Mysql 数据库(mysqldump) #

概述 #

本文档将指导您如何在 Windows 环境下使用mysqldump工具自动备份 MySQL 数据库。mysqldump是一个 MySQL 自带的命令行工具，用于备份数据库。通过设置 Windows 任务计划程序，可以实现定期自动备份，并保持最近 7 天的记录，删除 7 天之前的备份文件。

准备工作 #

在开始之前，请确保以下条件已满足：

  1. 已安装 MySQL 数据库。
  2. 具有执行mysqldump命令的权限。
  3. 已配置好 MySQL 的环境变量，或者知道 MySQL 的安装路径。

编写备份脚本 #

首先，我们需要编写一个批处理脚本（.bat 文件），用于执行mysqldump命令并备份数据库。

  1. 打开文本编辑器（如 Notepad++），创建一个新的文本文件。
  2. 输入以下内容，根据实际情况修改数据库名称、用户名、密码和备份路径：
chcp 936
@echo off
echo =============================
echo ### mysql database backup bat
echo =============================

echo =========当前日期时间==========
set nowdate=%date:~0,4%%date:~5,2%%date:~8,2%%time:~0,2%%time:~3,2%%time:~6,2%
echo %nowdate%
echo =============================

echo ======进入到mysql安装目录======
C:
cd C:\server\mysql-5.7.16-winx64\bin
echo =============================

echo ====备份mysql文件到指定文件夹====
echo mysql文件备份中...
echo ============================= -P端口、-u账号、-p密码 secrecy数据库
mysqldump -P3410 -uroot -pgengzhaoji.3536 secrecy > "C:\server\MySQLCopy_File\secrecy_%nowdate%.sql"
echo MySQL 备份成功
echo =============================

echo 备份的sql文件
forfiles /p "C:\server\MySQLCopy_File" /s /m *.sql
echo =============================

rem 获取"D:\MySQLCopy_File"目录下的sql文件,并且删除7天前的文件
echo *********************
echo * 按时间删除7天前的sql文件 *
echo *********************
echo 删除操作前存在的sql文件
forfiles /p "C:\server\MySQLCopy_File" /s /m *.sql
echo =============================

rem 删除sql文件操作
forfiles /p "C:\server\MySQLCopy_File" /s /m *.sql /d -7 /c "cmd /c del /q /f @path"

echo 删除操作后存在的sql文件
forfiles /p "C:\server\MySQLCopy_File" /s /m *.sql
echo =============================

rem 不关闭窗口操作
rem pause>nul 表示命令窗口不会出现“请按任意键继续”
rem pause>nul

rem 关闭窗口
@echo 数据库备份操作完成，5秒后关闭程序...
ping /n 5 127.1 >nul
exit
  1. 将文件保存为backup_mysql.bat。

设置 Windows 任务计划程序 接下来，我们将使用 Windows 任务计划程序来定期执行备份脚本。

  1. 打开“任务计划程序”（可以通过搜索“任务计划程序”或在控制面板中找到）。
  2. 在右侧操作栏中，点击“创建任务”。
  3. 在“常规”选项卡中，输入任务的名称和描述。
  4. 在“触发器”选项卡中，点击“新建”，设置备份的频率（例如每天、每周等）和具体时间。
  5. 在“操作”选项卡中，点击“新建”，选择“启动程序”，然后浏览并选择刚才创建的backup_mysql.bat文件。
  6. 在“条件”和“设置”选项卡中，根据需要进行设置。
  7. 点击“确定”保存任务。

验证备份 #

为了确保备份任务正常运行，可以手动运行一次任务，检查备份文件是否成功生成。

  1. 打开“任务计划程序库”，找到并右键点击刚才创建的任务，选择“运行”。
  2. 检查备份目录中是否生成了新的备份文件。
