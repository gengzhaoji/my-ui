Git & Gerrit 常用命令大全 #

本文档整合常规 Git 操作和 Gerrit 代码审查工作流的常用命令，适用于使用 Gerrit 的团队协作场景。

基础命令 #

命令说明
git init初始化本地仓库
git clone <远程仓库URL>克隆远程仓库（自动关联 origin）
git add .添加所有修改到暂存区
git commit -m "提交信息"提交暂存区的修改
git status查看工作区和暂存区状态

分支管理 #

命令说明
git checkout -b <新分支名>创建并切换到新分支
git branch -a查看所有分支（含远程分支）
git rebase origin/main将当前分支变基到 main 分支
git branch -D <分支名>强制删除本地分支

代码提交与审查 #

1. 提交到 Gerrit 审查队列 #

# 推送当前分支到 Gerrit 审查队列（关键命令！）
git push origin HEAD:refs/for/<目标分支>

# 示例：推送到 main 分支的审查队列
git push origin HEAD:refs/for/main

2. 安装 Gerrit Commit-msg 钩子 #

# 下载 Gerrit 提供的 commit-msg 钩子（自动生成 Change-Id）
curl -Lo .git/hooks/commit-msg http://<gerrit-url>/tools/hooks/commit-msg
chmod +x .git/hooks/commit-msg

3. 提交信息规范 #

提交信息中必须包含 Change-Id（由钩子自动生成）：

feat: Add new feature

This is a detailed description.

Change-Id: Ixxx8ad157f4d6f1443a1c2b7e33d8f4b6a7d8e5f

4. 更新补丁集（Patchset） #

# 1. 修改代码后提交（保留相同 Change-Id）
git commit --amend

# 2. 推送更新到同一审查请求
git push origin HEAD:refs/for/<目标分支>

远程仓库操作 #

命令说明
git fetch origin拉取远程仓库元数据（不自动合并）
git pull --rebase变基方式拉取代码（避免合并提交）
git remote add gerrit <URL>添加 Gerrit 远程仓库
git push gerrit --delete <分支>删除 Gerrit 远程分支

撤销与回退 #

命令说明
git reset --soft HEAD^回退提交但保留修改到暂存区
git revert <提交ID>创建撤销提交（Gerrit 审查后推荐使用）
git checkout -- <文件>丢弃工作区文件修改

实用技巧 #

1. 查看 Gerrit 变更 #

# 查看远程 Gerrit 变更
git ls-remote gerrit refs/changes/*

# 拉取特定补丁集
git fetch gerrit refs/changes/45/12345/1 && git checkout FETCH_HEAD

2. 解决冲突后重新推送 #

# 1. 拉取最新代码并变基
git pull --rebase origin main

# 2. 解决冲突后继续变基
git rebase --continue

# 3. 强制推送到 Gerrit（生成新补丁集）
git push origin HEAD:refs/for/main --force

3. 快速配置 Gerrit 推送 #

# 设置默认推送目标为 Gerrit 审查队列
git config remote.origin.push HEAD:refs/for/main

注意事项 #

  1. 禁止直接推送到分支
    Gerrit 通常禁止 git push origin main，必须通过 refs/for/ 提交审查。

  2. Change-Id 必须保留
    提交信息中的 Change-Id 是 Gerrit 追踪补丁集的关键，不要手动删除。

  3. 使用 rebase 代替 merge
    保持提交历史线性，避免出现合并提交（Merge Commit）。

  4. 谨慎使用 --force
    强制推送会覆盖 Gerrit 上的历史记录，仅在处理冲突后必要时使用。

  5. 审查通过后合并
    Gerrit 审核通过后，代码会由维护者合入目标分支，开发者无需手动操作。
