乾坤微前端架构 #

以下是关于 乾坤微前端架构 的详细知识点总结，包含主应用与子应用的核心实现示例：

一、乾坤微前端的核心原理 #

1. 沙箱机制 #

乾坤通过沙箱隔离子应用的 JS 和 CSS 环境，防止全局污染。

  • JS 沙箱：

    • 快照沙箱 (SnapshotSandbox)
      通过记录子应用加载前后的 window 对象差异，卸载时还原全局状态。
      适用场景：单实例子应用（如后台管理系统）。

      class SnapshotSandbox {
    constructor() {
        this.proxy = window; // 代理对象
        this.modifyPropsMap = {}; // 修改的属性
    }
    active() {
        this.windowSnapshot = {}; // 快照
        for (const prop in window) {
            this.windowSnapshot[prop] = window[prop];
        }
        // 恢复之前修改的属性
        Object.keys(this.modifyPropsMap).forEach((prop) => {
            window[prop] = this.modifyPropsMap[prop];
        });
    }
    inactive() {
        for (const prop in window) {
            if (window[prop] !== this.windowSnapshot[prop]) {
                this.modifyPropsMap[prop] = window[prop]; // 记录变更
                window[prop] = this.windowSnapshot[prop]; // 还原
            }
        }
    }
}

    • 代理沙箱 (ProxySandbox)
      使用 Proxy 代理 window，子应用的全局变量操作仅在代理对象中生效。
      适用场景：多实例并行子应用（如多标签页应用）。

      class ProxySandbox {
    constructor() {
        const rawWindow = window;
        const fakeWindow = {};
        this.proxy = new Proxy(fakeWindow, {
            get(target, prop) {
                return target[prop] || rawWindow[prop];
            },
            set(target, prop, value) {
                target[prop] = value;
                return true;
            },
        });
    }
}

  • CSS 隔离：

    • 动态样式表：子应用的样式在加载时插入 <style> 标签，卸载时移除。
    • Shadow DOM：通过 attachShadow 创建独立 DOM 树（需手动配置）。

2. 应用加载与生命周期 #

乾坤通过子应用暴露的生命周期钩子控制加载、挂载和卸载流程。

子应用代码示例（React 子应用）：

// 子应用入口文件（src/index.js）
import React from 'react';
import ReactDOM from 'react-dom';
import App from './App';

// 暴露乾坤生命周期函数
if (window.__POWERED_BY_QIANKUN__) {
    window.__webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__;
}

function render(props) {
    const { container } = props;
    ReactDOM.render(<App />, container ? container.querySelector('#root') : document.querySelector('#root'));
}

// 独立运行时直接渲染
if (!window.__POWERED_BY_QIANKUN__) {
    render({});
}

// 生命周期钩子
export async function bootstrap() {
    console.log('子应用启动');
}

export async function mount(props) {
    console.log('子应用挂载', props);
    render(props);
}

export async function unmount(props) {
    const { container } = props;
    ReactDOM.unmountComponentAtNode(container ? container.querySelector('#root') : document.querySelector('#root'));
}

3. 路由劫持 #

乾坤劫持 history.pushState 和 popstate 事件，根据 URL 匹配子应用的 activeRule 规则。

主应用路由配置示例：

// 主应用配置（main-app/src/setup.js）
import { registerMicroApps, start } from 'qiankun';

registerMicroApps([
    {
        name: 'react-sub-app',
        entry: '//localhost:7100',
        container: '#subapp-container',
        activeRule: '/react',
        props: { user: 'Admin' }, // 向子应用传递数据
    },
    {
        name: 'vue-sub-app',
        entry: '//localhost:7101',
        container: '#subapp-container',
        activeRule: '/vue',
    },
]);

// 启动乾坤
start({ prefetch: true }); // 预加载子应用资源

二、主应用与子应用通信 #

1. Props 通信 #

主应用通过注册子应用时传递 props，子应用在 mount 生命周期中接收。

主应用传递数据：

registerMicroApps([
    {
        name: 'sub-app',
        entry: '//localhost:7100',
        props: {
            onGlobalStateChange: (state) => console.log('状态变化:', state),
            setGlobalState: (state) => {
                /* 更新全局状态 */
            },
        },
    },
]);

子应用接收数据：

export async function mount(props) {
    props.onGlobalStateChange((state) => {
        console.log('收到全局状态:', state);
    });
    props.setGlobalState({ user: 'New User' });
}

2. 全局状态管理 #

使用 initGlobalState 实现主应用与子应用的状态共享。

主应用初始化全局状态：

// 主应用（main-app/src/state.js）
import { initGlobalState } from 'qiankun';

const initialState = { user: 'Guest' };
const actions = initGlobalState(initialState);

// 监听状态变化
actions.onGlobalStateChange((state, prevState) => {
    console.log('全局状态变更:', prevState, '→', state);
});

// 更新状态
actions.setGlobalState({ user: 'Admin' });

export default actions;

子应用使用全局状态：

export async function mount(props) {
    // 监听状态变化
    props.onGlobalStateChange((state) => {
        console.log('子应用收到状态:', state);
    });

    // 更新状态
    props.setGlobalState({ user: 'SubApp User' });
}

3. 事件总线（Event Bus） #

通过自定义事件实现跨应用通信。

主应用实现事件总线：

// 主应用（main-app/src/eventBus.js）
class EventBus {
    constructor() {
        this.events = {};
    }
    emit(eventName, data) {
        if (this.events[eventName]) {
            this.events[eventName].forEach((fn) => fn(data));
        }
    }
    on(eventName, fn) {
        this.events[eventName] = this.events[eventName] || [];
        this.events[eventName].push(fn);
    }
}

export const eventBus = new EventBus();

子应用监听与触发事件：

export async function mount(props) {
    // 监听主应用事件
    props.eventBus.on('refresh', (data) => {
        console.log('收到刷新指令:', data);
    });

    // 向主应用发送事件
    props.eventBus.emit('subapp-loaded', { status: 'ready' });
}

三、注意事项与最佳实践 #

  1. 公共依赖处理
    通过 Webpack externals 避免重复加载 React/Vue 等库：

    // 子应用 webpack 配置
module.exports = {
    externals: {
        react: 'React',
        'react-dom': 'ReactDOM',
    },
};

  2. 静态资源路径
    子应用需配置 publicPath，避免资源加载路径错误：

    // 子应用入口文件
if (window.__POWERED_BY_QIANKUN__) {
    __webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__;
}

  3. 子应用卸载清理
    在 unmount 生命周期中销毁定时器、事件监听等：

    export async function unmount() {
    // 清理定时器
    clearInterval(this.timer);
    // 移除事件监听
    window.removeEventListener('resize', this.handleResize);
}

  4. 性能优化

    • 启用预加载：start({ prefetch: true })
    • 子应用懒加载：动态 import() 非核心子应用

四、总结 #

适用场景：

  • 大型企业级应用（如后台管理系统、ERP）
  • 多团队协作开发（不同团队使用不同技术栈）
  • 旧系统渐进式迁移（逐步替换遗留模块）

核心优势：

  • 技术栈无关性：主应用与子应用可分别使用 React、Vue、Angular 等框架。
  • 独立部署：子应用构建后，主应用无需重新发布。
  • 安全隔离：JS/CSS 沙箱避免全局污染。
