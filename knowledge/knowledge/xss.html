防止 XSS 攻击 #

在前端富文本编辑器中防止 XSS 攻击需要综合策略，以下是关键步骤和示例：

1. 输入过滤（前端核心防御） #

使用 白名单机制 过滤 HTML 标签和属性，推荐使用成熟库：

import DOMPurify from 'dompurify';

// 只允许特定标签和属性
const cleanHTML = DOMPurify.sanitize(userInput, {
    ALLOWED_TAGS: ['p', 'strong', 'em', 'a', 'img'],
    ALLOWED_ATTR: ['href', 'src', 'alt', 'class'],
    FORBID_ATTR: ['style', 'onerror'],
    FORBID_TAGS: ['script', 'iframe'],
    ALLOWED_URI_REGEXP: /^(?:(?:https?|mailto):|[^#]+#)/i, // 禁止javascript:等协议
});

2. 输出转义（避免二次污染） #

插入 DOM 时确保内容安全：

// 正确：使用清理后的HTML
document.getElementById('content').innerHTML = cleanHTML;

// 错误示例：直接插入未过滤内容
// document.getElementById('content').innerHTML = userInput; // 高风险！

3. 内容安全策略（CSP） #

通过 HTTP 头限制资源加载：

Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted.cdn.com; style-src 'self' 'unsafe-inline'; img-src 'self' data:;
  • 禁止内联脚本 (unsafe-inline) 和 eval
  • 仅允许指定域名资源

4. 后端二次验证 #

前端过滤不可靠，需后端双重检查：

# Python示例：使用Bleach库
import bleach

allowed_tags = ['p', 'a', 'img']
allowed_attrs = {'a': ['href'], 'img': ['src']}
clean_html = bleach.clean(user_input, tags=allowed_tags, attributes=allowed_attrs)

5. 富文本编辑器配置 #

选择安全配置（以 TinyMCE 为例）：

tinymce.init({
    selector: '#editor',
    plugins: 'link image',
    valid_elements: 'p,strong,em,a[href|target],img[src|alt]',
    // 禁用危险功能
    paste_as_text: true, // 粘贴时转为纯文本
    paste_block_drop: true, // 阻止拖放非文本内容
});

6. 链接和动态内容防护 #

检查 URL 协议：

function sanitizeURL(url) {
    if (url.startsWith('javascript:') || url.startsWith('data:')) {
        return '#blocked';
    }
    return url;
}

7. 其他注意事项 #

  • 避免dangerouslySetInnerHTML（React）：仅在内容清理后使用。
  • 禁用危险 CSS：过滤expression()等动态样式。
  • 定期更新依赖库：确保 DOMPurify 等库为最新版本。
  • 测试 XSS 向量：使用工具如XSS Cheat Sheet验证防御。

总结 #

  • 核心原则：不信任用户输入，前后端双重过滤。
  • 工具优先：使用 DOMPurify、Bleach 等库而非手写正则。
  • 深度防御：结合 CSP、编辑器配置、URL 检查等多层防护。
